Ollama Setup Guide

Ollama runs AI models directly on your computer — no API key needed. First install Ollama, then allow this extension to reach it, download a model, and verify it is running.

Step 1 — Download and install Ollama

Download Ollama from the official site, run the installer, and complete the installation before continuing.

Download Ollama

After installing, open Ollama once so it can start its local service on your computer.

Step 2 — Allow extension access (CORS)

By default Ollama only accepts browser requests from localhost, not Chrome extensions. You need to set one environment variable and restart Ollama.

Windows
  1. Press Win + R, type SystemPropertiesAdvanced, press Enter
  2. Click Environment Variables
  3. Under "User variables" click New
  4. Name: OLLAMA_ORIGINS   Value: chrome-extension://*
  5. Click OK on all dialogs
  6. Right-click the Ollama icon in the system tray → Quit
  7. Reopen Ollama from the Start menu
macOS
  1. Open Terminal and run:
    launchctl setenv OLLAMA_ORIGINS "chrome-extension://*"
  2. Click the Ollama icon in the menu bar → Quit
  3. Reopen Ollama from Applications

Alternative — if you start Ollama from Terminal, prefix the command:
OLLAMA_ORIGINS="chrome-extension://*" ollama serve

Step 3 — Download a model

Ollama needs at least one AI model downloaded. Open a terminal (or Command Prompt on Windows) and run one of these:

ollama pull llama37B — good general use, needs ~4 GB RAM
ollama pull qwen2.5:3b3B — faster, works on lower-spec machines
ollama pull mistral7B — strong quality, good for writing tasks

After downloading, go back to AI Provider settings and use Detect installed models, or select Custom and type the model name exactly as shown by ollama list.

Step 4 — Verify it is running

Open your browser and go to http://localhost:11434. If you see "Ollama is running" the address is correct and Ollama is ready.

Troubleshooting

  • "Ollama blocked the request (CORS)" — the environment variable was not set, or Ollama was not fully restarted after setting it. Repeat Step 2.
  • "Model not found" — the model has not been downloaded yet, or the name does not match exactly. Run ollama list in a terminal to see installed models.
  • "Could not reach Ollama" — the app is not running. Open Ollama and check for its icon in the system tray (Windows) or menu bar (Mac).
  • Using a remote or non-localhost endpoint — this extension can only reach Ollama at localhost or 127.0.0.1. If you point the endpoint at another machine or IP address (e.g. http://192.168.1.10:11434), Chrome will block the request because the extension does not have permission for arbitrary IP addresses. To use Ollama, it must be running on the same computer as Chrome.
